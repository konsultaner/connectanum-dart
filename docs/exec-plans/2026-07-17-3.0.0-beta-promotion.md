# Exec Plan: 3.0.0 Beta Promotion

Status: complete; beta line promoted to `master`

## Goal

Promote the completed router, MCP, final WAMP profile, packaging, and benchmark
work from `add-router` to `master` as a coordinated `3.0.0-beta` prerelease.

## Release Contract

- Every versioned Dart package and Rust crate uses `3.0.0-beta`.
- Every public workspace dependency edge uses `^3.0.0-beta`.
- Package changelogs and runtime self-identification expose beta status.
- The strict package release gate rejects synchronized-version drift.
- No pub.dev publish tag is pushed until package ownership, first-publish
  requirements, and trusted publishing are confirmed by an authorized operator.
- `master` moves only after local verification and exact-head hosted checks are
  green. GitLab `master` is updated only when that move is a fast-forward.

   dry-run, and WAMP profile benchmark evidence.
4. Fast-forward `master` on eligible remotes and run the strict deployment-chain

5. Preserve release-relevant behavior from any divergent default branch before
   joining its history to the beta line.
