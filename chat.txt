Aufgabe Rust:

Shallow‑Parsing‑Strategie:

WAMP‑Nachrichten sind top‑level Arrays [Type, …]. Wir „laufen“ nur so weit, bis wir die Argument‑Teile erreicht haben:

z. B. CALL: [CALL(48), Request|id, Options|dict, Procedure|uri, Arguments?|list, ArgumentsKw?|dict]

INVOCATION / RESULT / EVENT analog.

Dabei dekodieren wir primär kleine Headerfelder (IDs, URIs), nicht Arguments/ArgumentsKw. Für diese bestimmen wir Byte‑Offsets und Längen im Ursprungsbuffer.

Optional erzeugen wir einen Head‑Blob (mit demselben Serializer), der alle Felder bis vor Arguments/ArgumentsKw enthält. Dieser ist klein (IDs/Options/URIs) und kommt via FFI direkt als Uint8List rüber.

Ergebnis:

MsgMeta { msg_type, serializer, request_id, ... }
HeadBlob { ptr,len }             // klein
PayloadDesc { handle, args(off,len), kwargs(off,len) }


Damit kann Dart sofort routen (per msg_type, request_id, URI …) und hat ein Handle auf die Payload ohne Kopie.

4) Speicher- & Lifetime‑Modell (ohne Kopieren bis Dart)

Jeder eingehende Frame liegt in einem Arc‑gepufferten Block (z. B. Bytes/Vec<u8>).

Für Dart erzeugen wir ExternalTypedData‑Views via Pointer<Uint8>.asTypedList(len) → kein Copy, du musst Lifetimes sichern. Seit Dart 3.1 können wir dabei direkt einen Finalizer registrieren.
Dart API Docs

Auf Dart‑Seite kannst du Payloads zwischen Isolates mit TransferableTypedData ohne weitere Kopie verschieben.

8) Dart‑Seite kurz (damit serializePayload() klar ist)

Du erzeugst ein NativeMessage aus native_msg_t:

class NativeMessage {
  final int connId, msgType, serializer;
  final int requestId, auxId;
  final Uint8List head;              // klein
  final int payloadHandle;
  final _Slice args, kwargs;

  Uint8List? serializePayload({bool kwargs=false}) {
    final which = kwargs ? 1 : 0;
    final ptrLen = _ffi.payload_get_view(payloadHandle, which);
    if (ptrLen.isEmpty) return null;
    final list = ptrLen.ptr.asTypedList(ptrLen.len,
        finalizer: _ffi.finalizerPtr, token: _ffi.token);
    // => ExternalTypedData, keine Kopie; Finalizer ruft payload_release(handle)
    return list;
  }
}


asTypedList liefert einen View auf native Speicher; die Lifetime sicherst du via Finalizer.

Wir definieren SniCertificate(hostname, certificateChainPem, privateKeyPem). Später kann die TLS‑Terminierung wahlweise „in native“ erfolgen (wenn tlsMode == TlsMode.native, übergibst du die PEMs an Rust für künftige kTLS/native TLS). Für den Listener‑MVP brauchen wir nur die Struktur & Tests.

maxRawSocketSizeExponent:
In RouterConfig.toNativeJson() enthalten; Validierung: mind. Spez‑Default (z. B. 16) und eine Obergrenze, Tests inklusive.

Abgrenzung Dart vs. Rust:
Auth/Message‑Security/Controller/Realm‑Maps bleiben komplett in Dart. Das Rust‑Backend bekommt nur, was es für den Transport und das spätere RawSocket‑Framing benötigt.
